# 邮件与登录

邮件与登录配置决定用户如何进入实例，以及系统如何发送验证和通知类邮件。

## 登录方式

当前实例可以使用账号密码和 OAuth 登录能力。OAuth 能力取决于部署时提供的客户端配置，例如 GitHub、Linux Do 或 Google。

如果某个 OAuth 服务没有配置客户端 ID 和密钥，前端不会把它作为可用登录方式展示。

## 邮件能力

邮件能力用于验证码、注册验证、密码重置或通知类场景。管理员需要配置 SMTP 或系统邮件服务。

配置完成后，应发送测试邮件验证发信地址、端口、安全连接和凭据是否正确。

## 注册治理

管理员可以控制注册是否开放、是否需要邀请码、是否允许绑定或解绑第三方账号。

建议公开实例关闭自由注册，使用邀请码或管理员创建账号的方式控制入口。

## 常见问题

如果用户无法登录，先检查：

- 对应登录方式是否启用。
- OAuth 回调地址是否和服务商后台一致。
- 邮件验证码是否被拦截到垃圾箱。
- 系统时间是否准确。
- `APP_SECRET` 是否在多次部署间保持一致。
