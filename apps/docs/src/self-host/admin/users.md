# 用户管理

用户管理用于查看和维护实例内账号。

## 用户列表

后台用户页展示实例用户信息。管理员可以按关键字筛选用户，并查看用户的身份、账号状态和关联数据摘要。

## 用户身份

Lexora 使用全局 `User` 作为账号身份。协作邀请使用用户码，不依赖公开用户名或展示名。

展示名可以被用户修改，不能作为唯一身份凭据。

## 注册与邀请

实例可以通过后台登录治理能力控制注册方式。是否允许开放注册、是否需要邀请码、是否启用第三方登录，取决于管理员配置。

## 风险操作

对用户状态、登录方式和账号数据的调整会影响用户访问能力。执行风险操作前，请先确认目标账号和影响范围。
